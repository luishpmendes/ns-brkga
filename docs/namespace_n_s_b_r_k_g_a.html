Detailed Description

Namespace enclosing the entire NS-BRKGA library.

All types, free functions, enumerations, and the main algorithm class (NSBRKGA) reside in this namespace. The supporting PathRelinking sub-namespace holds enumerations specific to the path relinking procedure.

typedef std::vector<double> NSBRKGA::Chromosome

Chromosome representation: a vector of alleles in \([0,1)\).

Chromosome is a type alias for std::vector<double>. Each element (allele) represents a continuous random key in the unit interval \([0, 1)\), so a chromosome of length \(n\) defines a point in the \(n\)-dimensional unit hypercube \([0,1)^n\).

Double precision is used because single-precision floats may be insufficient for problems that require fine-grained rank distinctions among alleles (e.g., permutation-based decoders that resolve ties by value).

Warning
Do not change the size of the chromosome inside the decoder; the library assumes chromosome.size() == chromosome_size at all times.
See also
NSBRKGA::NSBRKGA for parameter and usage details.
NSBRKGA::Population for sorted access to chromosomes in a population.

enum NSBRKGA::BiasFunctionType
strong

Preset bias-function types for parent selection during mating.

The bias function \(f(r)\) assigns a selection weight to the \(r\)-th parent (1-indexed, sorted best-to-worst). A higher weight increases the probability that parent \(r\) contributes an allele to the offspring. All presets are positive and non-increasing in \(r\), favouring the best parents.

Value Formula Notes
CONSTANT \(\frac{1}{P}\) ( \(P\) = total parents) Uniform; no

preference. | | CUBIC | \(r^{-3}\) | Strong elite bias. | | EXPONENTIAL| \(e^{-r}\) | Very strong elite bias. | | LINEAR | \(\frac{1}{r}\) | Moderate bias. | | LOGINVERSE | \(\frac{1}{\log(r+1)}\) | Often best in practice. | | QUADRATIC | \(r^{-2}\) | Moderate-strong bias. | | SQRT | \(r^{-1/2}\) | Biases toward worse parents. | | CBRT | \(r^{-1/3}\) | Mild bias toward worse parents. | | CUSTOM | User-defined via setBiasCustomFunction(). | Must be positive non-increasing. |

◆ CrossoverType

enum NSBRKGA::CrossoverType
strong

Crossover operator applied during mating.

The crossover operator determines how the offspring allele at each gene position is derived from the selected parents. Both operators are biased by the BiasFunctionType applied to the sorted parent ranks, giving higher-ranked (better) parents greater influence.

Value Mechanism
ROULETTE Discrete. Each gene independently selects one parent via
roulette-wheel selection weighted by bias(rank), then copies
that parent's allele verbatim. This is the classical MP-BRKGA

| | crossover. | | GEOMETRIC| Continuous. Each gene computes a weighted average of all | | | parent alleles. The weight for parent at rank \(r\) is drawn | | | uniformly from \([\phi(r),\,\phi(r+1)]\) where \(\phi\) is | | | the cumulative bias function. Offspring alleles are in \([0,1)\)| | | by construction. |

See also
NsbrkgaParams::crossover_type
Enumerator
ROULETTE 

Discrete biased roulette crossover: for each gene, a single parent is chosen via roulette wheel weighted by bias_function(rank) and its allele is copied.

GEOMETRIC 

Biased geometric (weighted-average) crossover: for each gene, every parent contributes with a random weight drawn from Uniform(phi(r), phi(r+1)) and the offspring allele is the normalised weighted average of the parent alleles.

enum NSBRKGA::DistanceFunctionType
strong

Distance-function types used by path relinking to select sufficiently dissimilar chromosome pairs.

Before starting path relinking the algorithm measures the distance between candidate chromosome pairs. Pairs that are too close (below a threshold) are discarded so that relinking explores a non-trivial portion of the search space.

◆ DiversityFunctionType

enum NSBRKGA::DiversityFunctionType
strong

Preset diversity-function types for adaptive elite-set sizing.

The diversity function maps a set of elite chromosomes (represented as vectors of doubles) to a scalar diversity score. After non-dominated sorting, the algorithm adds individuals to the elite set as long as doing so increases the diversity score.

◆ readConfiguration()

INLINE std::pair<NsbrkgaParams, ExternalControlParams> NSBRKGA::readConfiguration ( const std::string &  filename)

Reads algorithm parameters from a configuration file.

The configuration file contains key-value pairs, one per line, in any order. Lines starting with # and blank lines are ignored. All keys are case-insensitive. Every key in NsbrkgaParams (except CROSSOVER_TYPE, which is optional for backward compatibility) and ExternalControlParams must be present.

Parameters
filenamepath to the configuration file.
Returns
A pair {NsbrkgaParams, ExternalControlParams} populated from the file contents.
Exceptions
std::fstream::failureif the file cannot be opened, contains an unknown token, a duplicate token, an unparseable value, or a required token is missing.
